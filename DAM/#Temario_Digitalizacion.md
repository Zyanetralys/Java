## TEMA 1: DIGITALIZACION EN LOS SECTORES PRODUCTIVOS

### 1.1 CONCEPTOS FUNDAMENTALES

DIGITALIZACION:
- Definicion: Utilizar tecnologias y herramientas para conseguir que los procesos empresariales sean mas eficientes y, por tanto, la empresa sea mas competitiva.
- Caracteristica clave: No varia los procedimientos, adapta procesos analogicos a su version digital.
- Ejemplo: Pasar facturas en papel a PDF firmado digitalmente; pedidos, inventarios y registros en formato electronico.

TRANSFORMACION DIGITAL:
- Definicion: Proceso interno que va mas alla de lo digital; busca generar mas valor a clientes, proveedores, empleados, accionistas, mediante uso de estrategias de datos.
- Objetivo final: Adaptarse al contexto actual en un mundo interconectado e hiperactivo.
- Ejemplo: Usar analytics para predecir demanda, personalizar ofertas y optimizar cadena de suministro.

PROCESOS COMUNES DE TRANSFORMACION DIGITAL:
1. Digitalizar documentos (desmaterializacion): Pasar de papel a digital (pedidos, facturas, inventarios, registros, correspondencia). Permite firma digital, entrega telematica, almacenamiento en nube.
2. Formacion y concienciacion de trabajadores: Capacitar empleados en nuevas tecnologias; ver el cambio como oportunidad de mejora.
3. Redes sociales: Canales de contacto directo con clientes; contenido planificado, de calidad y util.
4. Sostenibilidad: Integrar preocupacion por medio ambiente, cambio climatico y huella de carbono.
5. Optimizacion del talento humano: Utilizar y distribuir recursos de forma mas eficiente.

### 1.3 CUARTA REVOLUCION INDUSTRIAL (KLAUS SCHWAB, 2016)

DEFINICION: Periodo de transformacion tecnologica desde principios de 2010, acuñado por Klaus Schwab, fundador del Foro Economico Mundial.

CARACTERISTICAS PRINCIPALES:
1. Avances tecnologicos disruptivos: IA, IoT, robotica, VR/AR, nanotecnologia, computacion cuantica. Transforman forma de producir, trabajar y relacionarse.
2. Convergencia de tecnologias: Interaccion entre distintas tecnologias que se complementan para crear soluciones innovadoras.
3. Automatizacion y digitalizacion: Mayor eficiencia en produccion, reduccion costes/tiempos, mejora calidad y personalizacion.
4. Impacto en mercado laboral: Cambios en forma de trabajar; necesidad de nuevas competencias; reconfiguracion de puestos.
5. Interconexion global: Mayor conexion entre empresas, consumidores, gobiernos y sociedad; facilita colaboracion e innovacion global.
6. Sostenibilidad y responsabilidad social: Busqueda de soluciones con impacto positivo en medio ambiente y sociedad.

### 1.4 SISTEMAS CIBERFISICOS (CPS - CYBER-PHYSICAL SYSTEMS)

DEFINICION: Sistemas integrados de computacion (ordenadores) capaces de controlar y colaborar con sistemas fisicos (maquinas y dispositivos) utilizando redes de comunicacion.

EJEMPLOS DE CPS:
- Drones agricolas
- Vehiculos autonomos
- Viviendas domotizadas
- Instrumentos medicos asistidos en cirugia
- Protesis bionicas
- Sistemas en fabricas inteligentes (control produccion, monitorizacion energetica)

CAMPOS DE APLICACION: Agricultura, aeronautica, infraestructuras civiles, energia, medicina, defensa, transporte, ciudades inteligentes.

FUNCIONAMIENTO DE LOS CPS: TRES CAPAS

CAPA DE PERCEPCION:
- Funcion: Obtencion de datos del entorno fisico mediante sensores.
- Sensores: Dispositivos que detectan cambios (temperatura, humedad, presion, velocidad, posicion, iluminacion) y los convierten en señales interpretables.
- Objetivo: Monitorizacion constante para toma de decisiones en tiempo real.

CAPA DE TRANSMISION:
- Funcion: Comunicacion e intercambio de datos entre componentes del sistema; asegurar que informacion de sensores llegue a procesadores y comandos sean ejecutados por actuadores.
- Actuadores: Dispositivos que ejecutan acciones fisicas basadas en decisiones del CPS (encender motor, ajustar valvula, etc.).

FUNCIONES PRINCIPALES DE CAPA DE TRANSMISION:
1. Interconexion de componentes: Vinculo entre sensores, procesadores y actuadores; asegura funcionamiento coherente.
2. Gestion de red: Supervision de rendimiento, deteccion/correccion de problemas; ajuste parametros para mejorar eficiencia.
3. Sincronizacion y tiempos: Transmision sincronizada de datos, coherencia temporal para correcto funcionamiento.
4. Seguridad: Enrutamiento, cortafuegos, sistemas de deteccion/prevencion de intrusiones (IDS/IPS).

TECNOLOGIAS REQUERIDAS PARA CAPA DE TRANSMISION:
- Comunicacion:
  * Redes cableadas: Ethernet, fibra optica (alta velocidad, baja latencia)
  * Redes inalambricas: Wi-Fi, Bluetooth, Zigbee, LTE, 5G
  * Protocolos: MQTT, CoAP, OPC UA, HTTP
- Seguridad:
  * Cifrado: Proteccion de datos en transmision
  * Autenticacion y autorizacion: Acceso solo a usuarios/dispositivos autorizados
  * Calidad de servicio (QoS)
  * Latencia y ancho de banda minimos
- Confiabilidad:
  * Garantizar llegada de datos sin perdida y en orden correcto

CAPA DE APLICACION:
- Funcion: Procesar datos recibidos para tomar decisiones y ejecutar acciones.

FUNCIONES PRINCIPALES DE CAPA DE APLICACION:
1. Procesar informacion con algoritmos avanzados para decisiones informadas.
2. Administrar almacenamiento seguro de datos historicos para analisis retrospectivos.
3. Combinar datos de multiples fuentes para vision general del sistema.
4. Realizar funciones de seguridad y privacidad (encriptacion, proteccion, cumplimiento normativo).
5. Supervisar continuamente estado del sistema para detectar y responder a problemas.
6. Utilizar datos analizados para mejorar rendimiento y eficiencia, ajustando parametros.

TECNOLOGIAS REQUERIDAS PARA CAPA DE APLICACION:
- Algoritmos de toma de decisiones: Machine Learning y AI para predecir, optimizar y mejorar rendimiento.
- Control de actuadores: Emision de comandos ejecutados por actuadores fisicos; automatizacion de tareas.
- Interfaces de usuario: Paneles de control, apps moviles, sistemas SCADA (Supervisory Control and Data Acquisition).
- Servicios cloud: Almacenamiento, procesamiento intensivo, analisis avanzado; escalabilidad y flexibilidad.
- Middleware: Facilita comunicacion entre componentes y gestiona interoperabilidad.

### 1.5 ENTORNOS IT Y OT: DIFERENCIAS Y SEMEJANZAS

DEFINICIONES:
- IT (Tecnologias de la Informacion): Tecnologias para recopilar, procesar, almacenar y distribuir informacion. Importantes en administracion de recursos, toma de decisiones basadas en datos, estrategias empresariales. Tambien conocidas como digitalizacion en negocio.
- OT (Tecnologias de Operacion): Supervision y control directo de procesos fisicos y equipos en entornos industriales/produccion. Hardware y software para control y monitoreo de dispositivos y procesos industriales. Tambien conocidas como digitalizacion en planta.

COMPONENTES IT EN SISTEMAS PRODUCTIVOS:
1. Infraestructura hardware/software: Servidores, ordenadores, dispositivos de red, programas y aplicaciones para operaciones, recopilacion y procesamiento de datos.
2. Redes de comunicacion: LAN, WAN, tecnologias inalambricas, intranets, extranets para conectividad e intercambio de informacion.
3. Sistemas de gestion de datos: Bases de datos, sistemas de gestion de contenido, plataformas para almacenar, analizar y utilizar informacion.
4. Seguridad y proteccion de datos: Ciberseguridad, politicas de privacidad, cumplimiento normativo para garantizar integridad de datos.
5. Aplicaciones empresariales: ERP, CRM, herramientas de inteligencia empresarial.

HERRAMIENTAS IT MAS UTILIZADAS:
- Microsoft Azure: Plataforma cloud con computacion, analisis, almacenamiento, redes.
- Amazon Web Services (AWS): Almacenamiento, bases de datos, entrega de contenido, escalabilidad.
- Salesforce: Plataforma CRM basada en cloud para gestionar contactos, ventas, servicio al cliente.
- SAP: Software de aplicaciones empresariales para gestion de procesos y flujo de informacion.
- Microsoft 365: Word, Excel, PowerPoint, Outlook, OneDrive, Teams para productividad y colaboracion.
- Zoom: Videollamadas, conferencias web, reuniones en linea, mensajeria en grupo.

COMPONENTES OT EN SISTEMAS PRODUCTIVOS:
1. Sistemas de control industrial: PLC (controladores logicos programables), DCS (sistemas de control distribuidos), SCADA (adquisicion de datos y control supervisado) para control automatizado en tiempo real.
2. Sensores y dispositivos de campo: Sensores, actuadores y dispositivos que recogen informacion del entorno (luz, temperatura, presion) para actuar sobre procesos industriales.
3. Interfaces hombre-maquina (HMI): Interfaces que facilitan la actuacion de operadores humanos con sistemas de control.
4. Redes industriales y de comunicacion: Redes especificas para conectar dispositivos de tecnologias de operacion.
5. Seguridad operativa: Medidas para garantizar seguridad fisica y cibernetica del sistema.

RELACION ENTRE IT Y OT:
| INTERACCION | BENEFICIO |
|-------------|-----------|
| Interconexion | Gestion empresarial mas eficiente mediante convergencia de tecnologias |
| Gestion de datos | IT trabaja con datos de empresa/operaciones; OT con datos de produccion/rendimiento |
| Colaboracion | Equipos de ambas tecnologias trabajan juntos para correcta implementacion tecnologica |
| Mejora de eficiencia | IT mejora gestion/proceso de datos; OT mejora produccion/operaciones |
| Flexibilidad en produccion | Conexion permite adaptar procesos a demandas cambiantes del mercado |
| Mejor experiencia del cliente | Digitalizacion permite personalizar productos/servicios y comunicacion mas agil |
| Seguridad integral | Trabajo conjunto para minimizar y eliminar riesgos ciberneticos |

### 1.7 EVOLUCION: EMPRESA CLASICA A EMPRESA DIGITALIZADA

VENTAJAS DE LA TRANSFORMACION DIGITAL:
1. Mayor agilidad en toma de decisiones: Procesos digitalizados + datos en tiempo real = decisiones mas rapidas y acertadas.
2. Mejora en eficiencia operativa: Automatizacion reduce errores, optimiza recursos, aumenta productividad.
3. Acceso a nuevos mercados y clientes: Presencia online amplia alcance global.
4. Mejora en experiencia del cliente: Personalizacion, servicios eficientes, adaptacion rapida a necesidades.
5. Reduccion de costes: Optimizacion de procesos, reduccion de gastos de infraestructura, mejora eficiencia energetica.
6. Innovacion continua: Fomento de creatividad para crear nuevos productos/servicios que respondan a demandas del mercado.

## TEMA 2: TECNOLOGIAS HABILITADORAS DIGITALES (THD)

### 2.1 DEFINICION DE THD

DEFINICION: Tecnologias que impulsan la innovacion y permiten a las empresas desarrollar el proceso de transformacion digital, proporcionando la tecnologia necesaria para adaptarse a la Industria 4.0.

CARACTERISTICAS COMUNES DE LAS THD:
| CARACTERISTICA | DESCRIPCION |
|---------------|-------------|
| Innovacion | Permiten creacion de nuevos modelos de negocio, servicios y productos |
| Interconexion | Se comunican entre si, multiplicando las opciones que ofrecen |
| Escalabilidad | Pueden crecer para adaptar su dimension a necesidades cambiantes |
| Versatilidad | Se pueden implementar en amplia variedad de empresas e industrias |
| Analitica avanzada | Ofrecen informacion en tiempo real y permiten analisis al instante para corregir decisiones |
| Personalizacion | Se adaptan a experiencias que demandan clientes y usuarios |
| Automatizacion | Muchas son autonomas en su cometido, mejorando beneficios empresariales |
| Seguridad | Incorporan medidas de proteccion avanzadas que garantizan integridad y confidencialidad |
| Ahorro | Optimizan uso de recursos, reducen costes operativos, minimizan desperdicios |

ATENCION: El proceso de transformacion digital NO requiere implementar todas las THD, ni hacerlo de forma acelerada. La clave: dar pequenos pasos, analizar resultados desde perspectiva empresarial, y avanzar adoptando mejoras sobre lo consolidado.

### 2.2 PRINCIPALES TECNOLOGIAS HABILITADORAS DIGITALES

2.2.1 REDES 5G:
- Definicion: Quinta generacion de tecnologia de redes moviles.
- Prestaciones:
  * Mayor velocidad de conexion (hasta 20 Gbps de descarga)
  * Latencia muy reducida (1-2 milisegundos)
  * Capacidad de gestionar mayor numero de dispositivos simultaneamente
  * Importante ahorro de energia vs tecnologias anteriores

2.2.2 COMPUTACION EN LA NUBE (CLOUD COMPUTING):
- Definicion: Servicios informaticos ofrecidos al usuario a traves de internet (almacenamiento, procesamiento, bases de datos, software, analisis) sin mantener infraestructura fisica localmente.
- Proveedores principales: Amazon Web Services (AWS), Microsoft Azure, Google Cloud Platform.

2.2.3 TECNOLOGIAS DE PROCESAMIENTO MASIVO DE DATOS (BIG DATA):
- Definicion: Capacidad de recopilar, almacenar, analizar y mostrar patrones/tendencias de grandes volumenes de datos para extraer informacion buscada.
- Origen de datos: Transacciones comerciales, sensores, redes sociales, registros en linea, etc.
- Tipos de datos:
  * Estructurados: Datos en bases de datos relacionales (tablas con campos definidos)
  * Semiestructurados: Datos en formatos como XML o JSON (etiquetas sin esquema rigido)
  * No estructurados: Texto, imagenes o videos (correos electronicos, fotos, videos de redes sociales)
- Herramientas para manejar Big Data: Sistemas de almacenamiento distribuido, procesamiento paralelo, algoritmos de analisis, aprendizaje automatico, visualizacion de datos.

2.2.4 CIBERSEGURIDAD:
- Definicion: Procesos y tecnologias disenados para proteger sistemas informaticos de amenazas ciberneticas.
- Pilares fundamentales:
  * Prevencion: Firewalls, antivirus, sistemas de deteccion de intrusiones, controles de acceso, parches de seguridad.
  * Deteccion: Monitorizacion permanente de red, deteccion de anomalias, analisis de registros, sistemas SIEM (Security Information and Event Management).
  * Respuesta: Planes y procedimientos para responder a incidentes: movilizacion de equipos, contencion de amenazas, investigacion forense digital, recuperacion de sistemas.
  * Recuperacion: Copias de seguridad, reparacion de sistemas comprometidos, implementacion de controles adicionales.
  * Educacion y concienciacion: Formacion de usuarios en mejores practicas: contrasenas seguras, identificacion de phishing, uso seguro de dispositivos y redes.

2.2.5 BLOCKCHAIN:
- Definicion: Tecnologia de registro distribuido que permite mantener una base de datos segura y transparente sin intermediarios.
- Funcionamiento: Datos agrupados en bloques enlazados secuencialmente formando cadena inmutable; cada bloque contiene informacion del bloque anterior -> garantiza integridad y autenticidad.
- Aplicaciones: Criptomonedas (Bitcoin), logistica, votacion electronica, gestion de identidades.
- Registro distribuido: Base de datos compartida y sincronizada entre varios nodos de una red (vs bases de datos centralizadas controladas por un unico servidor).

2.2.6 INTELIGENCIA ARTIFICIAL (IA):
- Definicion: Capacidad de maquinas y sistemas informaticos para realizar tareas como aprender de la experiencia, resolver problemas, comprender/procesar lenguaje natural, adaptarse a nuevas situaciones, reconocer patrones.
- Evolucion: Aprendizaje automatico (machine learning) -> aprendizaje profundo (deep learning).

2.2.7 REALIDADES INMERSIVAS:
- Realidad Virtual (VR):
  * Definicion: Sumerge al usuario en entorno completamente generado por ordenador.
  * Dispositivos: Visores o gafas de VR.
  * Caracteristica: Usuario puede interactuar en entorno virtual.
- Realidad Aumentada (AR):
  * Definicion: Combina elementos del mundo real con elementos generados por ordenador, superponiendo informacion digital en entorno fisico.
  * Dispositivos: Smartphones, tabletas con camara y software especializado.
  * Caracteristica: Graficos, texto o elementos interactivos superpuestos en vista del mundo real.

2.2.8 ROBOTICA COLABORATIVA (COBOTICA):
- Definicion: Enfoque de robotica donde robots y seres humanos trabajan juntos de forma segura y eficiente en entornos productivos.
- Diferencia con robotica tradicional: Robots colaborativos interactuan directamente con humanos en espacio compartido (vs robots industriales que operan en zonas aisladas por seguridad).

2.2.9 GEMELOS DIGITALES:
- Definicion: Replica virtual o digital de objeto, sistema o proceso del mundo real, utilizada para realizar analisis y simulaciones que serian costosos, peligrosos o impracticables en la realidad.
- Construccion: Integracion de multiples fuentes de informacion (datos historicos, algoritmos de analisis y prediccion) procesados y visualizados mediante software especializado.
- Funcion: Mostrar estado actual y comportamiento que tendra el objeto/sistema en entorno real.

2.2.10 IMPRESION 3D (FABRICACION ADITIVA):
- Definicion: Proceso de fabricacion que construye objetos tridimensionales, capa por capa, a partir de datos digitales.
- Ventajas:
  * Capacidad de crear geometrias complejas
  * Personalizacion rapida y economica
  * Reduccion de desperdicios de material
  * Produccion de componentes con propiedades especificas y optimizadas

2.2.11 INTERNET DE LAS COSAS (IOT):
- Definicion: Tecnologia que permite que dispositivos fisicos (maquinaria industrial, sistemas domesticos como riego o electrodomesticos) se conecten y comuniquen a traves de internet.
- Funcionamiento: Dispositivos recopilan y generan datos de su entorno y propio funcionamiento.
- Objetivo: Automatizar y optimizar procesos; crear nuevos servicios mas inteligentes y eficientes.

### 2.4 MEJORAS CON LA IMPLEMENTACION DE LAS THD POR SECTOR

2.4.1 SECTOR SANITARIO:
- Telemedicina y atencion remota: Consultas a distancia, reduccion de visitas fisicas, mayor acceso en areas rurales.
- Inteligencia artificial: Diagnostico temprano de enfermedades, analisis de imagenes medicas, prediccion de brotes epidemiologicos.
- Big Data: Recopilacion y analisis de datos de salud para identificar tendencias, mejorar gestion de enfermedades, desarrollar terapias.
- Genomica y medicina de precision: Tratamientos personalizados basados en perfil genetico del paciente; mayor eficacia, reduccion de efectos secundarios.
- Realidades inmersivas: Formacion medica, planificacion quirurgica, rehabilitacion de pacientes mediante experiencias inmersivas.
- Robotica en cirugia: Procedimientos menos invasivos, mayor precision del cirujano, menor tiempo de recuperacion del paciente.
- Automatizacion de procesos: Eficiencia en trabajos administrativos (llegada de pacientes, facturacion, gestion de inventarios).
- Wearables y dispositivos de monitorizacion: Monitorizacion en tiempo real de signos vitales, actividad fisica, parametros de salud.
- Aplicaciones moviles de salud: Seguimiento de estado de salud, recordatorios de visitas/medicaciones, acceso a informacion medica fiable.

2.4.2 SECTOR SERVICIOS:
- Automatizacion y eficiencia operativa: RPA (Robotic Process Automation) reduce tiempo y costes de tareas repetitivas/administrativas.
- Inteligencia artificial: Chatbots y asistentes virtuales para atencion al cliente; combinada con Big Data para experiencias personalizadas.
- Cloud Computing: Almacenamiento y gestion de datos eficiente y segura; facilita colaboracion online, agilidad y escalabilidad.
- Trabajo remoto y colaboracion digital: Videoconferencias, plataformas de gestion de proyectos, suites de productividad para flexibilidad laboral.
- Seguridad cibernetica mejorada: Proteccion de datos sensibles de clientes y operaciones criticas frente a ataques.
- Logistica: Digitalizacion de cadena de suministro, drones y vehiculos autonomos para optimizar gestion y distribucion de inventarios.

2.4.3 SECTOR INDUSTRIAL:
- Automatizacion y robotica: Robots y sistemas automatizados para mayor precision y eficiencia en lineas de produccion; robots colaborativos para seguridad laboral.
- Internet de las Cosas: Sensores y dispositivos para monitorizacion en tiempo real de maquinaria; mantenimiento predictivo, reduccion de tiempos de inactividad.
- Fabricacion aditiva (Impresion 3D): Produccion rapida y economica de prototipos y piezas personalizadas; reduccion de costes y tiempos de desarrollo.
- Blockchain: Mejora de trazabilidad en cadena de suministro, transparencia e integridad de datos desde materia prima hasta producto final.
- Realidades inmersivas: Optimizacion de formacion de empleados, planificacion y simulacion de procesos productivos, mantenimiento de maquinaria mediante guias visuales.
- Gemelos digitales: Replicas virtuales de equipos y procesos fisicos para simulacion y analisis en tiempo real; deteccion de problemas antes de que ocurran.

## TEMA 3: CLOUD Y SISTEMAS CONECTADOS

### 3.1 CLOUD COMPUTING

3.1.1 QUE ES LA NUBE:
- Definicion: Servicio de almacenamiento y gestion de datos a traves de internet, operado por entramado de servidores remotos ubicados por todo el mundo.
- Permite a usuarios acceder desde cualquier lugar con dispositivo con conexion a Internet a gran variedad de recursos.
- Servicios ofrecidos: Almacenamiento de datos, acceso a bases de datos, aplicaciones, programas informaticos, software, streaming de video, redes sociales.

FUNCIONAMIENTO BASICO:
1. Usuario utiliza dispositivo (ordenador, smartphone, tablet) para conectarse a internet (red fija o inalambrica).
2. Proveedor de internet lleva datos desde dispositivo hasta servidor de destino, utilizando direccion IP o direccion web.
3. Servidor procesa datos, opera con la informacion y devuelve una respuesta.

EJEMPLOS DE USO COTIDIANO:
- "Subir" album de fotos a la nube: Guardar informacion en servidor alojado en cualquier parte del mundo.
- Correo electronico (Gmail): Conectarse desde dispositivo, servidor de Google procesa solicitud, consulta correo nuevo, devuelve respuesta visible en pantalla. Datos en nube = acceso desde cualquier dispositivo.

3.1.2 NIVELES DE CLOUD COMPUTING:

PROVEEDORES DE SERVICIO EN LA NUBE: Empresas que ofrecen servicios de computacion en la nube, permitiendo almacenar, gestionar y procesar datos y aplicaciones a traves de internet en lugar de servidores locales.

NIVELES DE PRESTACION DE SERVICIOS:
1. Infraestructura como servicio (IaaS):
   - Ofrecen: Infraestructura como servidores, redes y almacenamiento a traves de Internet.
   - Usuarios: Acceden a recursos y los utilizan segun necesidad, pagando solo por lo que consumen.
   - Nivel mas alto de servicio.
   - Proveedores principales: Amazon Web Services (AWS), Google Cloud Platform (GCP), Microsoft Azure.

2. Plataforma como servicio (PaaS):
   - Ofrecen: Plataforma de procesamiento completa que permite crear, probar y desplegar aplicaciones sin gestionar infraestructura subyacente.
   - Incluye: Entornos de desarrollo, herramientas de base de datos, middleware.
   - Nivel medio de servicio.
   - Proveedores principales: Heroku, Google App Engine, Microsoft Azure.

3. Software como servicio (SaaS):
   - Ofrecen: Aplicaciones y software bajo suscripcion.
   - Usuarios: Acceden a aplicaciones desde cualquier dispositivo con conexion a internet.
   - Nivel mas cercano al usuario final.
   - Proveedores principales: Google Workspace (antes G Suite), Microsoft Office 365, Salesforce.

MODELOS DE DESPLIEGUE:
- Nube publica: Comparte recursos y ofrece servicios al publico a traves de internet.
- Nube privada: No se comparte; ofrece servicios a traves de red interna privada, normalmente hospedada en entorno local.
- Nube hibrida: Comparte servicios entre nubes publicas y privadas, segun finalidad.

### 3.2 POSIBILIDADES DEL TRABAJO EN LA CLOUD

ALMACENAMIENTO Y GESTION DE DATOS:
- Almacenamiento de archivos: Guardar grandes cantidades de datos de forma segura y accesible desde cualquier lugar. Proveedores: Amazon S3, Google Cloud Storage, Microsoft Azure Blob Storage.
- Bases de datos en la nube: Gestionar bases de datos sin hardware propio. Proveedores: Amazon RDS, Google Cloud SQL, Azure SQL Database.

DESARROLLO Y PRUEBAS DE SOFTWARE:

- CI/CD (Integracion y Despliegue Continuos): Automatizar flujo de trabajo de desarrollo con herramientas como Jenkins, GitHub Actions, GitLab CI.

INTELIGENCIA ARTIFICIAL Y MACHINE LEARNING:
- Modelado y entrenamiento: Crear y entrenar modelos de IA y ML. Proveedores: Google AI Platform, AWS SageMaker, Azure Machine Learning.
- APIs de IA: Implementar soluciones de reconocimiento de voz, procesamiento de lenguaje natural, vision por computadora. Proveedores: Google Cloud, IBM Watson, Azure Cognitive Services.

APLICACIONES WEB Y MOVILES:
- Aplicaciones web: Desplegar aplicaciones web. Proveedores: AWS Elastic Beanstalk, Google App Engine, Azure App Service.
- Backend para aplicaciones moviles: Gestionar backend incluyendo autenticacion, bases de datos en tiempo real, notificaciones push. Proveedor: Firebase de Google.

INFRAESTRUCTURAS:
- Maquinas virtuales: Crear y gestionar maquinas virtuales en la nube. Proveedores: Amazon EC2, Google Compute Engine, Azure Virtual Machines.

PLATAFORMAS:
- Desarrollo de aplicaciones: Desarrollar, desplegar y gestionar aplicaciones sin infraestructura subyacente. Plataformas: Heroku, AWS Elastic Beanstalk, Google App Engine.

HERRAMIENTAS PARA EMPRESAS:
- Productividad: Mejorar productividad con aplicaciones como G Suite, Office 365, Slack.
- Gestion de relaciones con clientes (CRM): Mejorar ventas y gestion de relaciones con clientes. Soluciones: Salesforce, HubSpot.
- Backup y recuperacion de desastres: Implementar soluciones de backup automatizadas para asegurar datos criticos y recuperacion rapida en caso de fallo. Proveedores: AWS Backup, Google Cloud Storage, Azure Backup.
- Servicios de seguridad: Medidas de proteccion como cortafuegos, deteccion de intrusiones, cifrado de datos. Proveedores: AWS Shield, Google Cloud Security, Azure Security Center.

STREAMING Y CONTENIDOS MULTIMEDIA:

IOT (INTERNET DE LAS COSAS):

### 3.3 EDGE COMPUTING

3.3.1 QUE ES EL EDGE COMPUTING:
- Definicion: Modelo de computacion distribuida que acerca el analisis y almacenamiento de datos lo mas cerca posible de donde se generan, practicamente en el propio dispositivo.
- Funciones similares a cloud computing, pero procesamiento mas cercano = mayor velocidad y reduccion de tiempo de respuesta o latencia.
- Latencia: Tiempo en que la informacion tarda en ir al servidor y volver al usuario.

3.3.2 RELACION DEL EDGE COMPUTING CON LA NUBE:
- Al enviar informacion a procesar a centros de datos mas cercanos, edge computing disminuye tiempo de respuesta respecto a cloud computing.
- Edge computing complementa cloud computing al proporcionar capacidades de procesamiento adicionales y mas cercanas al usuario final.
- Nube: Tareas que requieren grandes cantidades de datos, almacenamiento a largo plazo, procesamiento intensivo.
- Edge computing: Aplicaciones que requieren respuestas rapidas y en tiempo real (IoT, analisis de datos en tiempo real, realidad aumentada, vehiculos autonomos, telemedicina).
- En edge computing, informacion diversificada en multiples dispositivos = datos mas repartidos = reduccion de danos de posible ciberataque.
- Procesar datos en borde de red reduce cantidad de datos transferidos a nube = optimizacion de ancho de banda y reduccion de costes de transferencia.

3.3.3 APLICACIONES DEL EDGE COMPUTING:
- Internet de las Cosas (IoT): Dispositivos inteligentes en hogares, ciudades y fabricas que recopilan y procesan datos localmente para decisiones en tiempo real.
- Smart cities: Eficiencia de ciudad inteligente (redes inteligentes, vehiculos autonomos, alumbrado publico, transporte publico) depende de flujo masivo de datos; descentralizacion evita colapsos en sistema.
- Fabricas inteligentes: Datos generados por maquinaria favorecen procesos de produccion mas inteligentes, mejor interaccion entre cobots y humanos, prevencion de accidentes, minimizacion de danos.
- Monitoreo de salud remota: Dispositivos medicos que analizan datos de pacientes en tiempo real para alertar de situaciones criticas inmediatamente.
- Aplicaciones de realidad aumentada y realidad virtual: Favorecen prestacion de servicios a distancia como asistencia sanitaria o formacion.
- Videojuegos: Requieren latencia muy reducida para permitir respuestas rapidas de jugadores.

### 3.4 EDGE, FOG Y MIST: RELACION CON LA NUBE

|--------|-----------|-----------|---------------|-------------|

## TEMA 4: LA INTELIGENCIA ARTIFICIAL

### 4.1 TIPOS DE IA

4.1.1 CLASIFICACION EN FUNCION DE TAREAS ESPECIFICAS:

INTELIGENCIA ARTIFICIAL DEBIL (NARROW AI):
- Definicion: Sistemas disenados y entrenados para realizar tareas especificas. Enfocada en unica tarea o conjunto limitado de tareas; no puede llevar a cabo otras fuera del ambito para el que fueron disenadas.
- Ejemplos:
  * Asistentes virtuales: Siri, Alexa, Google Assistant (responder preguntas, establecer recordatorios, encender luces).
  * Sistemas de recomendacion: Netflix, Amazon, Spotify (sugieren productos/contenido basado en preferencias y comportamientos del usuario).
  * Sistemas de filtrado de spam en correo electronico.
  * Algoritmos de reconocimiento de voz o facial: Identificar o verificar persona en funcion de caracteristicas biometricas.

INTELIGENCIA ARTIFICIAL FUERTE (AGI - ARTIFICIAL GENERAL INTELLIGENCE):
- Definicion: Tipo de inteligencia que tendria capacidad de entender, aprender y aplicar conocimientos de manera general (forma similar a ser humano).
- Estado actual: Concepto teorico, no existe en practica. Investigaciones en IA todavia se centran en desarrollar y perfeccionar sistemas de IA debil.
- Caracteristicas clave de IA fuerte:
  * Aprendizaje autonomo: Capacidad de aprender de manera autonoma a traves de experiencia, sin programacion especifica para cada tarea; generar conocimiento nuevo a partir de datos y experiencias.
  * Razonamiento: Capacidad de razonar, planificar y resolver problemas complejos no previamente definidos o estructurados.
  * Emocion y consciencia: En teoria, capacidad de comprender y simular emociones humanas (aspecto que sigue siendo tema de debate y especulacion).

4.1.2 CLASIFICACION EN FUNCION DE COMO SE REPRESENTA Y MANIPULA LA INFORMACION:

INTELIGENCIA ARTIFICIAL SIMBOLICA:
- Definicion: Se basa en manipulacion de simbolos y reglas para representar y procesar conocimiento. Utiliza representaciones explicitas y formales de conceptos y relaciones existentes entre ellos empleando redes semanticas, ontologias y sistemas basados en reglas.
- Caracteristicas principales:
  * Representacion explicita del conocimiento: Hechos, reglas y axiomas se codifican en lenguajes formales como logica proposicional, logica de primer orden, lenguajes especificos.
  * Razonamiento basado en reglas: Sistemas simbolicos utilizan reglas para inferir nueva informacion a partir de datos existentes (reglas tipo "si-entonces" para tomar decisiones o resolver problemas).
  * Transparencia y comprensibilidad: Por su representacion explicita, sistemas simbolicos son mas facilmente interpretables y comprensibles por humanos; se puede seguir razonamiento paso a paso y entender como se llega a conclusion.
  * Manipulacion de lenguajes naturales: Inicialmente utilizada para tareas como comprension y generacion de lenguajes naturales, utilizando gramaticas formales y otros metodos simbolicos.
- Limitaciones: Gestion de grandes cantidades de datos y toma de decisiones en tiempo real.

INTELIGENCIA ARTIFICIAL SUBSIMBOLICA:
- Definicion: Se basa en modelos matematicos y metodos computacionales que intentan emular aprendizaje y procesamiento de informacion de manera similar a como lo hacen sistemas biologicos, particularmente cerebro humano.
- Caracteristicas principales:
  * Redes neuronales artificiales: Inspiradas en estructura y funcionamiento del cerebro; compuestas por unidades interconectadas (neuronas artificiales) que procesan informacion en paralelo. Capaces de aprender patrones y realizar tareas como clasificacion, reconocimiento de imagenes, procesamiento de lenguaje natural.
  * Aprendizaje profundo (deep learning): Utiliza multiples capas (a veces miles) de neuronas artificiales para extraer conclusiones de alto nivel de datos complejos. Gran exito en tareas como reconocimiento de imagenes, traduccion automatica, generacion de texto.
  * Algoritmos geneticos: Inspirados en teoria de evolucion; utilizan operaciones similares a seleccion natural, cruce y mutacion para evolucionar soluciones a problemas complejos.
  * Sistemas basados en agentes: Usan multiples agentes autonomos que interactuan entre si y con entorno para resolver problemas. Cada agente actua en funcion de reglas simples, pero interaccion combinada alcanza comportamientos complejos.
  * Logica difusa (fuzzy logic): Va mas alla de valores binarios (verdadero o falso); utiliza diferentes grados de verdad o falsedad para manejar incertidumbre e imprecision en toma de decisiones.
- Ventaja: Manejar grandes cantidades de datos y adaptarse a nuevas situaciones sin programacion explicita y detallada de reglas.
- Desventaja: Baja interpretabilidad y transparencia; dificil entender exactamente como y por que sistema subsimbolico llega a determinada solucion o conclusion.

### 4.2 EVOLUCION DE LA IA

CRONOLOGIA HISTORICA:

DECADAS DE 1940 Y 1950: PRIMEROS CONCEPTOS
- 1943: Warren McCulloch y Walter Pitts proponen modelo de neuronas artificiales (primeros trabajos en redes neuronales).
- 1950: Alan Turing publica "Computing Machinery and Intelligence"; introduce concepto de Test de Turing para evaluar inteligencia de maquina.

DECADAS DE 1950: NACIMIENTO FORMAL DE LA IA
- 1956: Conferencia de Dartmouth; John McCarthy, Marvin Minsky, Nathaniel Rochester y Claude Shannon acunan termino "inteligencia artificial" y proponen investigacion colaborativa en campo.
- Finales de decada de 1950: Primeros estudios sobre busqueda automatica y resolucion de problemas.

DECADAS DE 1960 Y 1970: PRIMEROS ALGORITMOS Y SISTEMAS
- 1966: Programa ELIZA (Joseph Weizenbaum); primer intento de procesamiento de lenguaje natural.
- Decadas de 1960 y 1970: Desarrollo de sistemas expertos: DENDRAL (interpretacion de datos quimicos); MYCIN (diagnostico medico).
- 1972: Aparece Prolog, lenguaje de programacion asociado especificamente con IA simbolica.

DECADAS DE 1980: AUGE DE SISTEMAS EXPERTOS Y RETROPROPAGACION
- Decada de 1980: Sistemas expertos se comercializan ampliamente, pero muchos fracasan por limitadas capacidades de hardware y rigidez.
- 1986: David Rumelhart, Geoffrey Hinton y Ronald Williams popularizan algoritmo de retropropagacion para entrenamiento de redes neuronales; revitaliza interes en redes neuronales artificiales.

DECADAS DE 1990: EXPANSION DE IA Y APRENDIZAJE AUTOMATICO
- 1997: Deep Blue (IBM) derrota a campeon mundial de ajedrez Garry Kasparov; gran hito en proceso.
- Decada de 1990: Expansion de IA y profundizacion de avances de aprendizaje automatico; investigacion en algoritmos de aprendizaje automatico se expande; comienzan a surgir modelos de aprendizaje estadistico y tecnicas de mineria de datos.

DECADAS DE 2000: AVANCES EN HARDWARE Y BIG DATA

### 4.3 LA INTELIGENCIA ARTIFICIAL Y LOS DATOS

4.3.1 MINERIA DE DATOS (DATA MINING):
- Definicion: Proceso de descubrir patrones, tendencias y relaciones significativas en grandes conjuntos de datos mediante uso de tecnicas estadisticas, matematicas y de inteligencia artificial.
- Implica analisis de datos desde diferentes perspectivas y conversion de estos datos en informacion util para toma de decisiones.

ETAPAS DEL PROCESO DE MINERIA DE DATOS:
1. Recoleccion y preparacion de datos: Recopilacion de datos de diversas fuentes; limpieza y preprocesamiento para eliminar inconsistencias y descubrir que puede faltar.
2. Transformacion de datos: Conversion de datos a formato adecuado para su analisis.
3. Modelado y analisis: Aplicacion de tecnicas de mineria de datos para construir modelos que representen patrones encontrados en datos.
4. Evaluacion e interpretacion: Evaluacion de modelos construidos para determinar precision y utilidad; interpretacion de resultados para extraer conclusiones valiosas.
5. Despliegue y uso de resultados: Implementacion de conocimientos obtenidos en aplicaciones practicas (mejora de procesos empresariales, toma de decisiones estrategicas).

TECNICAS EN MINERIA DE DATOS:
- Clasificacion: Asignar elementos a categorias predefinidas basandose en caracteristicas observadas.
- Clustering: Agrupar elementos en clusters o grupos basados en similitudes inherentes entre ellos.
- Regresion: Identificar relaciones entre variables y predecir valores continuos.
- Analisis de asociacion: Encontrar reglas de asociacion que describan relaciones significativas entre variables en datos.
- Deteccion de anomalias: Identificar datos que no se ajustan a patrones normales o esperados.

4.3.2 COMO TRABAJAN CONJUNTAMENTE IA Y BIG DATA:
- Tanto aprendizaje automatico (machine learning) como aprendizaje profundo (deep learning) necesitan gran cantidad de datos para aprender y mejorar de errores cometidos. Proceso de entrenamiento infinito con busqueda de mejora continua.

ENFOQUES DE APRENDIZAJE:
1. Aprendizaje supervisado:
   - Definicion: Objetivo es aprender a partir de datos etiquetados. Para cada entrada, modelo recibe etiqueta correspondiente.
   - Ejemplo: Banco con historial de transacciones etiquetadas como "fraudulentas" o "no fraudulentas". Algoritmo analiza transacciones etiquetadas para aprender caracteristicas relacionadas con fraude (transacciones inusualmente grandes, ubicaciones geograficas inesperadas). Despues de entrenar, modelo capaz de predecir y detectar posibles transacciones fraudulentas en tiempo real.

2. Aprendizaje no supervisado:
   - Definicion: Trabaja con datos que no estan etiquetados. Objetivo es encontrar patrones o estructuras ocultas en datos.
   - Aplicaciones comunes: Tecnica de agrupacion (clustering), reduccion de dimensionalidad.
   - Ejemplo: Datos de comportamiento de compra de usuarios en tienda online sin etiquetas de tipo de cliente. Algoritmo analiza patrones de compra y agrupa clientes en segmentos basados en comportamiento ("compradores frecuentes", "compradores ocasionales").

3. Aprendizaje por refuerzo:
   - Definicion: Se da en entorno donde se adoptan acciones para lograr recompensa. A diferencia de aprendizaje supervisado, no se dan respuestas correctas explicitas; se aprende a traves de sistema prueba/error, con retroalimentacion en forma de recompensas o castigos.
   - Ejemplo: Robot en fabrica que tiene que desplazarse de punto A a punto B evitando obstaculos. Algoritmo de aprendizaje por refuerzo permite a robot explorar entorno y aprender de interacciones, recibiendo recompensas por llegar a destino de manera eficiente y penalizaciones al chocar con obstaculos. Con tiempo, robot mejora rutas y capacidad para navegar de manera autonoma y eficiente.

### 4.4 LENGUAJES DE PROGRAMACION EN IA

- Popularidad: Mas popular debido a simplicidad y gran cantidad de bibliotecas y frameworks disponibles.

  * Soporte para paradigmas de programacion: Admite multiples paradigmas (programacion orientada a objetos, funcional, imperativa).

  * Facilidad para visualizacion de datos: Herramientas como Matplotlib, Seaborn y Plotly permiten crear visualizaciones de datos detalladas y personalizadas; facilitan analisis e interpretacion de resultados en proyectos de IA.

  * Entorno interactivo de desarrollo: Herramientas como RStudio proporcionan entorno interactivo que facilita experimentacion, desarrollo rapido y reproduccion de analisis.
  * Amplia comunidad y recursos: Comunidad activa y extensa ofrece soporte a traves de foros y documentacion variada; ayuda a resolver problemas y mejorar continuamente capacidades de R.

- Virtudes principales:
  * Rendimiento y escalabilidad: Maquina virtual (JVM) y optimizaciones de rendimiento hacen de Java opcion robusta para aplicaciones de IA a gran escala.
  * Bibliotecas y frameworks de IA: Varias bibliotecas y frameworks utiles para IA (Deeplearning4j, Weka, MOA, Apache Spark MLlib).
  * Seguridad: Caracteristicas de seguridad integradas que protegen aplicaciones de IA contra amenazas y vulnerabilidades.
  * Integracion con big data: Se integra bien con tecnologias de big data (Hadoop, Apache Spark) que son esenciales para manejar grandes volumenes de datos en proyectos de IA.

- Numero de unidades producidas cada dia: [150, 180, 160, 175, 170, 165, 190]

### 5.2 CICLO DE VIDA DEL DATO

DEFINICION: Conjunto de etapas por las que pasa un dato desde su creacion hasta su eliminacion en contexto de digitalizacion.

ETAPAS DEL CICLO DE VIDA:

1. CAPTURA O RECOPILACION:
   - Fuentes de datos: Bases de datos internas, sensores, encuestas, redes sociales, transacciones comerciales, aplicaciones moviles, etc.
   - Herramientas y tecnologias: Sistemas de gestion de bases de datos (DBMS), API, formularios en linea, dispositivos de internet de las cosas, etc.
   - API (Application Programming Interface): Conjunto de definiciones y protocolos que permiten a diferentes aplicaciones comunicarse entre si; actua como puente que facilita interaccion entre distintos sistemas, aplicaciones o plataformas, permitiendo compartir datos y funcionalidades de manera eficiente.

2. ALMACENAMIENTO:
   - Ubicaciones: Nube, servidores locales, bases de datos, entre otras.
   - Data warehouse: Sistema de almacenamiento de datos estructurados, disenado para facilitar analisis y toma de decisiones.
   - Data lake: Repositorio de almacenamiento disenado para almacenar grandes volumenes de datos en su forma bruta o semiestructurada.

3. PROCESAMIENTO Y ANALISIS DE LOS DATOS:
   - Limpieza: Datos recopilados a menudo contienen errores o duplicados que deben ser corregidos o eliminados.
   - Normalizacion: Datos deben ser convertidos a formato comun para asegurar coherencia, especialmente si provienen de fuentes diversas.
   - Transformacion: Datos "en bruto" se transforman en informacion util.

4. DISTRIBUCION:
   - Formas de distribucion: Reportes, dashboards, envio de alertas y notificaciones.
   - Dashboard (panel de gestion o cuadro de mandos): Interfaz en la que se reflejan, de forma grafica, los indicadores clave de desempeno (KPI).

5. USO DE LOS DATOS:
   - Toma de decisiones estrategicas y operativas.
   - Optimizacion de procesos (cadena de suministro, personalizacion de experiencia del cliente, etc.).
   - Desarrollo de estrategias.

6. ARCHIVADO, ELIMINACION O DESTRUCCION:
   - Archivado: Datos que ya no son de uso frecuente, pero deben permanecer disponibles por razones legales, historicas o de referencia; mover datos a sistemas de almacenamiento de menor costo, pero con suficiente capacidad de recuperacion.
   - Eliminacion segura: Si datos ya no son necesarios o no deben archivarse; puede implicar borrado fisico de discos duros, sobreescritura de datos o uso de software especializado para garantizar que datos no puedan ser recuperados.

### 5.3 ANALISIS DE DATOS Y BIG DATA

5.3.1 ANALISIS DE DATOS:

FASES DEL ANALISIS DE DATOS:

1. EXPLORACION DE DATOS:
   - Revision de datos para entender estructura y calidad.
   - Tecnicas de visualizacion para identificar tendencias o anomalias.
   - Analisis descriptivo: Entender caracteristicas basicas de datos (distribuciones, tendencias, patrones); herramientas como graficos, tablas y estadisticas descriptivas (calculo de medidas basicas como medias, medianas, desviaciones estandar y percentiles para resumir caracteristicas principales).
   - Identificacion de patrones: Mediante tecnicas como clustering, analisis de correlacion o identificacion de outliers, buscar patrones o relaciones dentro de datos.

2. MODELADO DE DATOS:
   - Creacion de modelos matematicos o estadisticos que describen relaciones entre variables y permiten efectuar predicciones.
   - Modelos predictivos: Utilizacion de tecnicas de machine learning para construir modelos basados en datos historicos que puedan predecir resultados futuros o identificar riesgos (regresion, arboles de decision, redes neuronales).
   - Analisis prescriptivo: Va mas alla de prediccion, sugiriendo posibles acciones a tomar con base en resultados obtenidos del analisis predictivo.
   - Simulacion: Creacion de escenarios hipoteticos para prever como podrian evolucionar datos bajo diferentes condiciones.

3. VALIDACION DE MODELOS:
   - Objetivo: Validar rendimiento.
   - Metodo: Probar precision y validez de modelos construidos utilizando conjuntos de datos de prueba o validacion; asegura que modelos sean robustos y generalizables.

4. INTERPRETACION DE LOS RESULTADOS:
   - Traducir resultados del analisis en insights accionables.
   - Implica no solo interpretar resultados estadisticos, sino tambien contextualizarlos dentro de entorno economico que nos interesa.

5. IMPLEMENTACION Y MONITOREO:
   - Despliegue de modelos: Integrar modelos predictivos y analiticos en sistemas operacionales para que puedan ser utilizados en tiempo real o en procesos empresariales.
   - Monitoreo continuo: Supervisar rendimiento de modelos a lo largo del tiempo para asegurar que sigan siendo precisos y relevantes; ajustar y recalibrar modelos en funcion de necesidades de cada momento.

6. VISUALIZACION Y COMUNICACION DE RESULTADOS:
   - Presentar hallazgos de manera clara y concisa a partes interesadas a traves de informes, dashboards o presentaciones; facilitar comprension y toma de decisiones.

5.3.2 BIG DATA:

DEFINICION: Infraestructura necesaria para almacenar, procesar y analizar grandes volumenes de datos de manera eficiente.

CARACTERISTICAS DEL BIG DATA (LAS 5 V):
1. Volumen: Capaz de gestionar gran cantidad de datos que, por complejidad y dinamismo, seria imposible procesar con bases de datos tradicionales.
2. Velocidad: Se caracteriza por velocidad a la que se generan, capturan y procesan datos.
3. Variedad: Datos son de indole muy diversa, provienen de gran variedad de fuentes (transacciones comerciales, sensores, redes sociales, registros en linea, etc.).
4. Veracidad: Los datos pueden ser estructurados (bases de datos relacionales), semiestructurados (XML, JSON) o no estructurados (texto, imagenes, videos).
5. Valor: Objetivo del analisis de big data es ofrecer datos funcionales y relevantes para toma de decisiones; capaz de extraer informacion, identificar patrones, tendencias y correlaciones, obtener conocimientos significativos en distintos ambitos.

HERRAMIENTAS PARA MANEJAR BIG DATA: Sistemas de almacenamiento distribuido, procesamiento paralelo, algoritmos de analisis de datos, aprendizaje automatico, visualizacion de datos.

5.3.3 RELACION ENTRE ANALISIS DE DATOS, BIG DATA E IA:

INTERRELACION:
- Ciencia del analisis de datos no seria posible sin gran volumen de datos que maneja big data (requiere gran volumen de informacion).
- Sin grandes cantidades de datos, algoritmos de inteligencia artificial no podrian aprender y generalizar con eficacia.

COMPARATIVA:

| ASPECTO | ANALISIS DE DATOS | BIG DATA | INTELIGENCIA ARTIFICIAL |
|---------|-----------------|----------|------------------------|
| Definicion | Proceso de inspeccionar, limpiar, transformar y modelar datos con objetivo de descubrir informacion util, sacar conclusiones y apoyar toma de decisiones. | Manejo y analisis de grandes volumenes de datos que son demasiado complejos, variados o de consumo inmediato para ser procesados por metodos tradicionales de analisis de datos. | Campo de informatica que se enfoca en crear sistemas que pueden realizar tareas que tipicamente requieren inteligencia humana (aprendizaje, percepcion, toma de decisiones, adaptacion). |
| Enfoque | Extraer insights y patrones a partir de datos estructurados y, en menor medida, no estructurados. | Infraestructura y tecnologias necesarias para manejar enormes conjuntos de datos; se ocupa de las 5 V: volumen, velocidad, variedad, veracidad, valor. | Emular inteligencia humana mediante algoritmos y modelos que permiten a maquinas realizar tareas como reconocimiento de voz, vision por computadora, toma de decisiones, etc. |
| Ejemplos | Informes de negocio, optimizacion de procesos, toma de decisiones basadas en datos historicos, analisis predictivo, personalizacion. | Analisis de datos masiva a gran escala, analisis de redes sociales, internet de las cosas, analisis en tiempo real. | Asistentes virtuales (Alexa, Siri), sistemas de recomendaciones, vehiculos autonomos, diagnosticos medicos automatizados, juegos y simulaciones. |

### 5.4 INGENIERIA DE DATOS

DEFINICION: Disciplina que se encarga de infraestructuras y sistemas que permiten gestion y analisis de grandes volumenes de datos. Se encuentra al principio de ciclo de vida de datos, afectando a captura, almacenamiento y preparacion de datos para posterior analisis.

RELACION CON CIENCIA DE DATOS:
- Ingenieria de datos es parte esencial dentro de campo mas amplio de ciencia de datos.
- Se encarga de creacion de sistemas que gestionan flujo de datos desde origen hasta consumo final por parte de aplicaciones analiticas, modelos de machine learning, etc.

ACTIVIDADES PRINCIPALES:
- Disenar y construir pipelines de datos, bases de datos y arquitecturas que soporten gestion y flujo continuo de datos en organizacion.
- Herramientas de ETL: Apache NiFi, Talend, Apache Airflow.
- Plataformas de big data: Hadoop, Spark.
- Bases de datos SQL y NoSQL.
- Tecnologias: Bases de datos distribuidas, procesamiento en tiempo real, arquitecturas de microservicios, sistemas de almacenamiento en nube.

DIFERENCIA CON MINERIA DE DATOS:
- Ingenieria de datos: Se centra en crear infraestructura y procesos necesarios para gestionar y mover datos eficientemente dentro de organizacion.
- Mineria de datos: Se ocupa de extraer valor y conocimiento a partir de datos utilizando avanzadas tecnicas analiticas.
- Ambos campos son complementarios y esenciales en cualquier estrategia de datos.

ETAPAS CLAVE DEL PROCESO DE INGENIERIA DE DATOS:

1. Recopilacion de datos: Reunir datos de diversas fuentes (bases de datos, archivos, dispositivos de internet de las cosas); asegurar que datos sean relevantes y de alta calidad.
2. Almacenamiento de datos: Elegir metodos y tecnologias para almacenar datos de manera eficiente y segura (bases de datos relacionales, almacenamiento en nube, bases de datos NoSQL, etc.).
3. Procesamiento de datos: Transformar y limpiar datos en bruto para hacerlos aptos para analisis; incluye eliminacion de duplicados, correccion de errores, normalizacion, otras tecnicas de preprocesamiento.
4. ETL (Extraccion, Transformacion y Carga): Datos son extraidos de fuentes originales, transformados segun necesidades de negocio (limpieza, normalizacion, agregacion), y luego cargados en sistemas de destino como almacenes de datos.
5. Integracion de datos: Combinar datos de diferentes fuentes para crear vista unificada; implica correspondencia de esquemas, alineacion de formatos, resolucion de conflictos de datos.
6. Analisis y modelado de datos: Aplicar tecnicas estadisticas y de aprendizaje automatico para extraer informacion valiosa de datos (analisis descriptivos, predictivos o prescriptivos).
7. Visualizacion de datos: Crear representaciones visuales de datos y resultados de analisis para facilitar comprension y toma de decisiones.
8. Mantenimiento y optimizacion: Asegurar que sistemas de datos se mantengan actualizados y funcionen de manera eficiente; monitorizacion continua y mejora de tecnologias utilizadas segun sea necesario.
9. Gobernanza de datos: Implementar politicas y procedimientos para garantizar calidad, integridad y seguridad de datos; incluye gestion de acceso, cumplimiento de regulaciones, definicion de roles y responsabilidades en gestion de datos.

### 5.5 ALMACENAMIENTO DE DATOS EN LA NUBE

5.5.1 FUNCIONAMIENTO DEL ALMACENAMIENTO EN LA NUBE:
- Definicion: Modelo en el que datos se guardan en servidores remotos, accesibles a traves de internet, en lugar de estar almacenados localmente en discos duros o servidores fisicos propios del usuario o empresa.
- Centros de datos: Gestionados por proveedores de servicios en nube como Amazon Web Services (AWS), Google Cloud, Microsoft Azure, entre otros.
- Caracteristicas de centros de datos: Equipados con gran cantidad de servidores de gran capacidad; datos se distribuyen y replican en multiples ubicaciones para garantizar disponibilidad y redundancia.

TIPOS DE ALMACENAMIENTO EN LA NUBE:
1. Almacenamiento de archivos:
   - Similar a almacenamiento en discos duros tradicionales, pero accesible desde cualquier dispositivo conectado a internet.
   - Ejemplos: Google Drive, Dropbox.

2. Almacenamiento de objetos:
   - Informacion almacenada en "objetos" que contienen datos, sus metadatos y un identificador unico.
   - Modelo mas indicado para almacenar grandes cantidades de datos no estructurados.
   - Ejemplo: Amazon S3.

3. Almacenamiento de bloques:
   - Datos divididos en bloques de tamano fijo, cada uno con su propia direccion.
   - Tipo comun para almacenamiento de bases de datos y maquinas virtuales.

ACCESO A DATOS:
- Usuarios acceden a datos a traves de internet, utilizando interfaces web, aplicaciones moviles o mediante API (interfaces de programacion de aplicaciones) que permiten integracion con otros sistemas o servicios.

SEGURIDAD:
- Proveedores implementan multiples capas de seguridad: cifrado de datos, firewalls, sistemas de deteccion de intrusiones.
- Ofrecen opciones de respaldo y recuperacion ante desastres; aseguran que datos no se pierdan en caso de fallos tecnicos o errores humanos.

5.5.2 BENEFICIOS DEL ALMACENAMIENTO EN LA NUBE:

VENTAJAS PRINCIPALES:
1. Accesibilidad global: Usuario puede acceder a datos desde cualquier lugar con conexion a internet.
2. Adaptabilidad: Se adapta a todo tipo de datos (estructurados, semiestructurados y no estructurados).
3. Escalabilidad y elasticidad: Usuario puede aumentar o disminuir capacidad de almacenamiento segun necesidades sin tener que comprar o configurar hardware adicional.
4. Modelo de pago por uso: Solo se paga por cantidad de almacenamiento que se consume; mas economico que mantener infraestructura propia, especialmente para pequenas empresas o usuarios individuales.
5. Integracion: Permite integracion con otras aplicaciones y servicios en nube: API y herramientas que facilitan migracion y gestion de datos.
6. Colaboracion: Ofrece herramientas que facilitan trabajo colaborativo (edicion y acceso compartido en tiempo real).

### 5.6 APLICACION EN LAS EMPRESAS DE LA CIENCIA DE DATOS

DEFINICION DE CIENCIA DE DATOS: Disciplina que combina tecnicas de varias areas (estadistica, matematicas, informatica, ingenieria de software, etc.) con objetivo de extraer informacion valiosa a partir de datos. En ambito empresarial, esa informacion permite resolver problemas complejos, tomar decisiones informadas y generar predicciones.

APLICACIONES POR AREA DE LA ORGANIZACION:

DIRECCION:
- Business Intelligence (BI): Implementacion de dashboards y herramientas de analisis para monitorizar KPI en tiempo real; ayuda a responsables a tomar decisiones informadas basadas en datos actuales y precisos.
- Pricing dinamico: Ajuste de precios dinamicamente en funcion de demanda, competencia y otros factores; maximizacion de ingresos y margenes.

DESARROLLO DE PRODUCTO:
- Mejora continua: Analisis de datos de uso y retroalimentacion de clientes para identificar oportunidades de mejora o desarrollo de nuevos productos que mejoren satisfaccion del cliente y competitividad en mercado.
- Pruebas A/B: Tecnicas de ciencia de datos para realizar experimentos controlados que permiten comparar diferentes versiones de producto o servicio para determinar cual es mas efectiva.

PRODUCCION:
- Mantenimiento predictivo: Sensores en maquinas generan datos en tiempo real que se analizan para predecir fallos o necesidades de mantenimiento; evitan costosos tiempos de inactividad.
- Logistica: Optimizacion de rutas de entrega, gestion de inventarios en tiempo real, mejora de eficiencia.

MARKETING:
- Segmentacion: Analisis de datos de clientes para segmentar bases de datos en grupos con comportamientos y preferencias similares; permite personalizacion en campanas de marketing.
- Recomendaciones: Plataformas como Amazon o Netflix utilizan algoritmos de recomendacion que analizan comportamiento pasado de usuarios para sugerir productos o contenido que puedan interesarles.

VENTAS Y ATENCION AL CLIENTE:
- Analisis de sentimiento: Analisis de opiniones y comentarios de clientes en redes sociales y encuestas para entender percepcion y mejorar productos o servicios.
- Chatbots: Basandose en ciencia de datos, IA se utiliza para desarrollar chatbots que pueden responder preguntas frecuentes y asistir a clientes 24/7; mejora experiencia y reduce costes de personal.

RECURSOS HUMANOS:
- Prevencion de rotacion: Analisis de datos sobre desempeno, satisfaccion y otros factores para predecir que empleados podrian estar en riesgo de dejar empresa y tomar medidas preventivas.
- Reclutamiento: Algoritmos de ciencia de datos ayudan a filtrar candidatos, analizar curriculums y predecir que perfiles son mas adecuados para ciertos puestos.

DEPARTAMENTO DE TI:
- Deteccion de ciberataques: Tecnicas de ciencia de datos para monitorizar patrones de trafico de red y detectar actividades anomalas que podrian indicar ciberataque.

### 5.7 IMPORTANCIA DE LA SEGURIDAD EN EL MANEJO DE DATOS

CONTEXTO: Muchas organizaciones manejan datos personales sensibles (informacion financiera, historial medico, datos de identificacion). Necesidad de proteger esos datos para cumplir con normativas de privacidad y evitar infracciones que podrian resultar en multas sustanciales y danos a reputacion.

ESTRATEGIAS COMUNES DE PROTECCION CONTRA ATAQUES CIBERNETICOS:

1. Firewalls y antivirus: Utilizar firewalls para controlar trafico entrante y saliente; antivirus actualizados para detectar y eliminar malware (programas malignos).
2. Respuesta a incidentes: Desarrollar plan de respuesta a incidentes que incluya pasos claros para contener, investigar y recuperarse de ataques ciberneticos.
3. Respaldo de datos: Realizar copias de seguridad regulares de datos importantes y almacenarlas en lugar seguro fuera de red.
4. Monitoreo de redes: Implementar sistemas de deteccion de intrusiones (IDS) y sistemas de prevencion de intrusiones (IPS) para monitorear y responder a actividades sospechosas en red.
5. Control de acceso: Limitar acceso a sistemas y datos solo a aquellos que necesitan acceder a ellos para realizar su trabajo.
6. Autenticacion: Implementar autenticacion de multiples factores (MFA) para proteger cuentas de usuario con capa adicional de seguridad.
7. Auditorias de seguridad: Realizar auditorias periodicas para identificar y remediar vulnerabilidades en infraestructura de TI.
8. Actualizaciones de software: Mantener todo software (sistemas operativos, navegadores, aplicaciones) actualizado con ultimos parches de seguridad.
9. Formacion: Capacitar a empleados sobre practicas seguras en linea (no hacer clic en enlaces sospechosos, no descargar archivos adjuntos desconocidos).

## TEMA 6: DESARROLLO DE UN PROYECTO DE TRANSFORMACION DIGITAL DE UNA EMPRESA

### 6.1 ANALISIS Y DIAGNOSTICO INICIAL

IMPORTANCIA: Fase fundamental que establece bases para todo proyecto de transformacion digital. Conocer a fondo situacion actual de empresa para identificar areas de mejora.

PASOS A SEGUIR:

1. EVALUAR ESTADO ACTUAL DE LA EMPRESA:
   - Inventario tecnologico: Hardware y software disponibles, infraestructura (red, servidores, etc.).
   - Estudio de procesos empresariales: Detectar posibles cuellos de botella, duplicacion de procesos u otras ineficiencias.
   - Habilidades digitales de trabajadores: Conocer habilidades y actitud hacia cambio tecnologico mediante encuestas y entrevistas.

2. IDENTIFICACION DE NECESIDADES Y OPORTUNIDADES:
   - Analisis cuantitativos: Datos historicos de ventas o servicios, otros KPI que permitan identificar tendencias y areas de bajo rendimiento.
   - Analisis cualitativos: Inputs de directivos, empleados, clientes y proveedores de empresa.

3. ANALISIS DE COMPETENCIA Y TENDENCIAS DEL MERCADO:
   - Competencia: Saber que hace competencia y evaluar sus estrategias tecnologicas.
   - Tendencias: Analizar sector economico en que se mueve empresa, profundizando en tecnologias emergentes (IA, blockchain, IoT, big data, etc.) y como pueden aplicarse para mejorar procesos internos y oferta de productos o servicios.

6.1.1 HERRAMIENTAS Y METODOS:
- Analisis PESTEL: Evaluacion de entorno macroeconomico (politico, economico, social, tecnologico, ecologico, legal) para ver como puede influir en transformacion digital.
- Analisis DAFO: Identificacion de fortalezas, debilidades, oportunidades y amenazas especificas de empresa respecto a digitalizacion.
- Mapeo de procesos (BPM): Utilizacion de herramientas de modelado de procesos empresariales para efectuar mapeo detallado.
- Encuestas y entrevistas: Herramientas como SurveyMonkey o Google Forms para recopilar informacion de empleados y clientes.

6.1.2 PRODUCTOS DE LA FASE DE ANALISIS Y DIAGNOSTICO INICIAL:
- Informe de evaluacion de situacion actual: Documento detallado que cubre todos los aspectos mencionados (tecnologia, procesos, cultura, competencia, tendencias, etc.).
- Matriz DAFO.
- Mapeo de procesos y flujos de trabajo: Diagramas y documentos que detallan procesos actuales y sus posibles mejoras.
- Presentacion ejecutiva: Documento o presentacion que resuma hallazgos clave para comunicar diagnostico a directivos responsables.

OBJETIVO FINAL DE FASE: Tener vision clara de estado actual de empresa y haber identificado areas que se van a ver implicadas en transformacion digital.

### 6.2 DEFINICION DE OBJETIVOS

IMPORTANCIA: Establecer direccion clara y medible del proyecto; permitir marcar a todos implicados la linea a seguir.

CARACTERISTICAS DE OBJETIVOS:
1. Claridad y enfoque: Proporcionar direccion clara para proyecto; asegurar que todos comprendan y trabajen hacia mismos objetivos.
2. Medir el exito: Establecer base solida para medir exito de proyecto a traves de KPI y metricas claras.
3. Alineacion y compromiso: Alinear a todos implicados y obtener compromiso para implementacion exitosa de transformacion digital.
4. Gestion de recursos: Comprobar y garantizar que recursos esten bien distribuidos y se empleen de manera eficiente para alcanzar objetivos.
5. Preparacion ante posibles desafios: Identificar y planificar posibles riesgos que puedan aparecer en camino; prever manera de solucionarlos para asegurar que proyecto siga adelante sin interrupciones significativas.

6.2.1 HERRAMIENTAS Y METODOS:
- Cuadro de mando integral: Para alinear actividades diarias con estrategia a largo plazo.
- OKR (Objectives and Key Results): Metodologia para definir y seguir objetivos y sus resultados clave.
- Encuestas y entrevistas: Herramientas para recabar inputs de personal de empresa y asegurarse de que metas definidas son relevantes para todos.
- Software de gestion de proyectos: Herramientas como Microsoft Project, Asana o Trello para organizar y seguir progreso de proyecto.

6.2.2 PRODUCTOS DE LA FASE DE DEFINICION DE OBJETIVOS:
- Documento de objetivos: Documento detallado que describe objetivos estrategicos, tacticos y operativos.
- KPI y metricas: Conjunto claramente definido de KPI y metricas que se utilizaran para medir exito de plan de transformacion digital.
- Plan de comunicacion: Estrategia y calendario de comunicacion para mantener informados a todas personas implicadas.
- Asignacion de recursos: Informacion detallada de todos recursos (humanos, financieros, tecnologicos, etc.) necesarios para cada fase de proyecto.
- Matriz de riesgos: Documento que recoja riesgos identificados y estrategias de actuacion previstas para mitigarlos.
- Acta de reuniones: Registro de reuniones y talleres en los que queden plasmadas decisiones y acuerdos alcanzados.

OBJETIVO FINAL DE FASE: Tener vision clara y compartida de objetivos de proyecto y como se medira exito; asegurar que todos involucrados esten alineados y enfocados en alcanzar objetivo comun.

### 6.3 PLANIFICACION ESTRATEGICA

ENFOQUE: Desarrollar plan de ejecucion de proyecto de transformacion digital; definir acciones especificas, plazos y recursos (de todo tipo) necesarios para llevarlas a cabo.

ELEMENTOS DE PLANIFICACION:
- Hoja de ruta: Distintas fases y tareas a desarrollar en cada una, cronogramas para ejecutar cada hito definido.
- Asignacion de responsabilidades y roles: Configurar equipo responsable de proyecto, animar cooperacion interdepartamental.
- Recursos detallados: Entrar al detalle de recursos requeridos para ejecutar plan de transformacion.

CARACTERISTICAS PRINCIPALES:
1. Claridad en proceso: Definir claramente todas tareas y fases de proyecto; asegurar que todos involucrados comprendan su rol y responsabilidad.
2. Asignacion eficiente de recursos: Garantizar que recursos humanos, financieros y tecnologicos esten asignados y gestionados de manera optima.
3. Reducir riesgos: Identificar y planificar para mitigar riesgos potenciales; asegurar que no se produzcan interrupciones en ejecucion de plan.
4. Comunicacion efectiva: Mantener a todos implicados informados y comprometidos; asegurar transparencia y alineacion con objetivos de proyecto.
5. Seguimiento y control: Establecer mecanismos claros para seguimiento de progreso y control de desviaciones; mantener proyecto dentro de plazos y presupuestos previstos.

6.3.1 HERRAMIENTAS Y METODOS:
- Diagramas de Gantt: Para planificacion y visualizacion de fases de proyecto y sus tareas.
- Metodos agiles (Scrum, Kanban): Para gestion de proyectos iterativos y flexibles.
- Software de gestion de proyectos: Herramientas como Microsoft Project, Asana, Trello o Jira para planificacion, asignacion de tareas y seguimiento.
- CPM (Critical Path Method): Para identificar tareas cruciales que determinan duracion total de proyecto.
- Matriz de riesgos: Para identificacion, evaluacion y mitigacion de riesgos.

6.3.2 PRODUCTOS DE LA FASE DE PLANIFICACION ESTRATEGICA:
- Hoja de ruta de proyecto: Documento detallado que incluye todas fases y tareas con sus respectivos plazos. Fases: preparacion, desarrollo de piloto, implementacion a escala mayor, evaluacion y optimizacion.
- Asignacion de responsabilidades: Documento que detalla roles y responsabilidades de cada miembro de equipo (gerente, especialistas en TI, usuarios finales, etc.).
- Plan de recursos:
  * Humanos: Capacitacion continua; designar responsables de cambio dentro de cada departamento para facilitar adopcion de nuevas tecnologias.
  * Financieros: Presupuesto detallado para cada fase de proyecto, con reserva para imprevistos y clara justificacion de cada gasto en relacion con objetivos.
  * Tecnologicos: Infraestructura tecnologica escalable que permite expansion futura (servidores, software, equipos de red, herramientas de gestion de proyectos).
- Plan de comunicacion: Estrategia detallada que incluye mensajes, medios y calendario de comunicaciones.
  * Comunicacion interna (directivos y trabajadores): Boletines, webinars, talleres, reuniones.
  * Comunicacion externa (clientes y proveedores): Comunicados de prensa, redes sociales, informes.
  * Eje central: Informacion de ventajas de plan de transformacion digital.

CONSIDERACIONES ADICIONALES:
- Escalabilidad futura: Preparar terreno para crecimiento sostenido; asegurar que sistema mantenga relevancia y eficiencia a lo largo del tiempo.
- Gestion del cambio: Crucial para conseguir exito en proyecto o transformacion organizacional; debe llevarse a cabo, con mayor o menor intensidad, en todas etapas de proceso.
- Decision de personas: Decidir que personas de empresa son mas indicadas para llevar a cabo gestion del cambio y convencerles de importancia de hacerlo de manera decidida.

### 6.4 SELECCION E IMPLEMENTACION DE LA TECNOLOGIA

IMPORTANCIA: Fase fundamental; tecnologia seleccionada sera plataforma sobre la que se construira transformacion digital de empresa.

PROCESO:
1. Identificar necesidades tecnologicas de empresa.
2. Investigar opciones que ofrece mercado (demostraciones de proveedores, pruebas piloto).
3. Llevar a cabo comparativas, evaluaciones tecnicas y financieras.
4. Garantizar seguridad de todo sistema.

FASES DE IMPLEMENTACION TECNOLOGICA:

1. DESPLIEGUE INICIAL:
   - Implementacion gradual: Realizar despliegue gradual para minimizar riesgos; comenzar con departamentos o procesos mas criticos.
   - Supervision y monitorizacion: Monitorizar de cerca progreso inicial para identificar y resolver cualquier problema rapidamente.

2. CONFIGURACION Y PERSONALIZACION:
   - Ajustes tecnicos: Configurar y personalizar soluciones tecnologicas para que se adapten a necesidades especificas de empresa.
   - Integracion con sistemas existentes: Asegurar integracion fluida y efectiva con sistemas ya existentes.

3. PRUEBAS Y VALIDACION:
   - Pruebas de funcionalidad: Realizar pruebas exhaustivas para asegurar que todos componentes funcionen correctamente y cumplan requerimientos.
   - Feedback de usuarios: Recoger retroalimentacion de usuarios y realizar ajustes segun sea necesario.
   - Test de aceptacion del usuario.
   - Auditorias de seguridad: Realizar auditorias para garantizar que se mantengan estandares de seguridad durante implementacion.

6.4.1 HERRAMIENTAS Y METODOS:
- Software de evaluacion de tecnologia: Gartner Magic Quadrant (identificar y comparar mejores soluciones de mercado); Forrester Wave (analisis detallado de capacidades de diversos proveedores).
- Plataformas de gestion de proyectos: Jira (gestionar tareas, prioridades y avance de implementacion); Microsoft Project (planificacion detallada de cronograma de implementacion).
- Pruebas de concepto: Herramientas como Sandbox Testing realizan pruebas en entornos aislados para asegurar que nuevas tecnologias funcionen correctamente sin afectar sistemas actuales.
- Seguridad de TI: Asegurar que nuevas tecnologias cumplan con requisitos de seguridad mediante auditorias previas al despliegue completo.

6.4.2 PRODUCTOS DE LA FASE DE IMPLEMENTACION TECNOLOGICA:
- Informe de evaluacion de tecnologias: Documento que detalla investigacion, pruebas y evaluacion de tecnologias consideradas.
- Matriz comparativa: Informe visual que compara diferentes opciones tecnologicas en funcion de criterios clave.
- Pruebas de concepto: Informe con resultados de pruebas iniciales en entornos controlados.
- Contratos con proveedores: Contratos negociados y aprobados que detallan terminos, condiciones y precios de toda implementacion tecnologica.
- Programa de capacitacion: Plan de formacion para empleados, incluyendo manuales, tutoriales y sesiones de entrenamiento.
- Plan de soporte continuo: Estrategia para soporte tecnico y mantenimiento regular una vez finalizada implementacion.

OBJETIVO FINAL DE FASE: Elegir soluciones correctas, evaluarlas exhaustivamente y ejecutarlas de manera estrategica; asegurar que organizacion no solo cumpla objetivos a corto plazo, sino que este bien posicionada para adaptarse y crecer en futuro.

### 6.5 DESPLIEGUE Y EJECUCION

IMPORTANCIA: Momento en que toda planificacion se convierte en realidad; ejecucion efectiva permite lograr objetivos de proyecto.

ETAPAS DE LA FASE:

1. Preparacion final: Revisar planes de proyecto; asegurarse de que todos recursos esten listos y disponibles; considerar realizacion de prueba piloto mas para identificar posibles problemas antes de despliegue total.

2. Coordinacion y comunicacion: Definir cronograma claro y asegurarse de que todos participantes lo conozcan; establecer equipo de control para monitorear despliegue y resolver problemas en tiempo real.

3. Implementacion tecnica: Instalar y configurar nuevo hardware o software; ajustar configuraciones segun sea necesario; migrar informacion a nuevos sistemas.

4. Capacitacion y soporte: Garantizar que todo personal este preparado para utilizar nuevas herramientas o sistemas; activar soporte tecnico durante periodo de transicion para resolver cualquier problema que surja.

5. Monitoreo y aseguramiento de calidad: Monitorear funcionamiento de sistema y cumplimiento de objetivos iniciales; garantizar correcto funcionamiento.

6. Retroalimentacion y ajustes: Recoger opiniones y comentarios de todos usuarios y partes interesadas que permitan hacer primeros ajustes en sistema.

7. Documentacion: Asegurar que toda documentacion tecnica y de usuario este actualizada para reflejar estado final de sistema o proceso implementado; documentar lecciones aprendidas durante despliegue para futuras referencias y mejoras en proximos proyectos.

8. Evaluacion y cierre: Comparar resultados obtenidos con objetivos iniciales establecidos para evaluar exito de despliegue; preparar informe final que detalle resultados de despliegue, problemas encontrados y como se resolvieron, recomendaciones para futuro.

### 6.6 EVALUACION Y OPTIMIZACION

ENFOQUE: Etapa que podria situarse fuera de fase de desarrollo de plan de digitalizacion, pero es vital para asegurar que objetivos iniciales se cumplan y para identificar oportunidades de mejora a lo largo de tiempo.

OBJETIVO: Analizar rendimiento e implementar mejoras constantes para maximizar efectividad y eficiencia de proyecto o sistema implementado.
